# Homomorphism_(Hard)

มีระบบตัวเลขแบบวนรอบตั้งแต่ `0` ถึง `n - 1` เมื่อบวกเลขแล้วได้ค่ามากกว่าหรือเท่ากับ `n` จะวนกลับมาโดยใช้ modulo `n`

ตัวอย่างเช่น ถ้า `n = 5`

```text
3 + 4 = 7
7 mod 5 = 2
```

ดังนั้นในระบบนี้ `3 + 4` จะได้ค่าเป็น `2`

นักวิจัยสร้างตารางแปลงรหัส `f` สำหรับเลขแต่ละตัว โดย `f[i]` หมายถึงผลลัพธ์หลังจากแปลงเลข `i`

ตัวอย่างเช่น

```text
n = 5
f = 0 2 4 1 3
```

หมายความว่า

```text
0 -> 0
1 -> 2
2 -> 4
3 -> 1
4 -> 3
```

ตาราง `f` ไม่จำเป็นต้องเป็นการสลับค่าหรือ permutation ค่าในตารางสามารถซ้ำกันได้ และไม่จำเป็นต้องมีค่าครบทุกตัวตั้งแต่ `0` ถึง `n - 1`

ตารางแปลงรหัสนี้จะถือว่า “รักษาการบวก” ถ้าไม่ว่าจะเลือกเลขสองตัวใด `a` และ `b` ผลลัพธ์ของสองวิธีต่อไปนี้ต้องเท่ากันเสมอ

1. บวกก่อน แล้วค่อยแปลง
2. แปลงก่อน แล้วค่อยบวก

เขียนเป็นสูตรได้ว่า

```text
f[(a + b) mod n] = (f[a] + f[b]) mod n
```

สำหรับทุกค่า `a` และ `b` ตั้งแต่ `0` ถึง `n - 1`

ถ้าสูตรนี้เป็นจริงสำหรับทุกคู่ ให้แสดง `true`  
ถ้ามีอย่างน้อยหนึ่งคู่ที่ทำให้สูตรนี้ไม่จริง ให้แสดง `false`

## ตัวอย่างการตรวจ

ให้

```text
n = 5
f = 0 2 4 1 3
```

ลองเลือก `a = 1` และ `b = 3`

วิธีที่ 1: บวกก่อน แล้วค่อยแปลง

```text
(1 + 3) mod 5 = 4
f[4] = 3
```

วิธีที่ 2: แปลงก่อน แล้วค่อยบวก

```text
f[1] = 2
f[3] = 1
(2 + 1) mod 5 = 3
```

ทั้งสองวิธีได้ค่า `3` เท่ากัน จึงผ่านสำหรับคู่นี้ แต่โจทย์ต้องตรวจทุกคู่ `a, b`

## ข้อมูลนำเข้า (Input)

| **ลำดับ** | **รายละเอียด** |
|----------|----------------|
| บรรทัดที่ 1 | จำนวนเต็ม `n` แทนจำนวนค่าที่อยู่ในระบบ โดยที่ 1 &le; n &le; 500 |
| บรรทัดที่ 2 | จำนวนเต็ม `f[0] f[1] ... f[n-1]` แทนตารางแปลงรหัส โดยที่ 0 &le; f[i] < n |

## ข้อมูลส่งออก (Output)

| **ลำดับ** | **รายละเอียด** |
|----------|----------------|
| บรรทัดที่ 1 | แสดง `true` ถ้าตารางแปลงรหัสรักษาการบวก modulo n หรือแสดง `false` ถ้าไม่รักษาการบวก modulo n |

ให้แสดงผลเป็นตัวพิมพ์เล็กทั้งหมด และไม่มีช่องว่างเกิน

| **ตัวอย่างข้อมูลนำเข้า** | **ตัวอย่างข้อมูลส่งออก** |
|--------------------------|--------------------------|

| 5<br>0 1 4 2 3 | false |
